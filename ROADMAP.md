# ROADMAP

Detailed preparation roadmap for finishing the full repository in a structured way.

## Stage 1 - Foundation (Weeks 1-4)

Topics:
- `01_Basics`
- `02_Arrays`
- `03_Strings`
- `04_Searching`
- `05_Sorting`

Focus:
- clean loops and conditions
- array indexing discipline
- string traversal patterns
- linear vs logarithmic search
- sorting fundamentals and stability intuition

Target:
- 120-140 problems
- speed target: easy questions within 12-20 mins

## Stage 2 - Core Data Structures (Weeks 5-8)

Topics:
- `06_Recursion`
- `07_Bit_Manipulation`
- `08_Linked_List`
- `09_Stack`
- `10_Queue`
- `11_Binary_Search`
- `12_Hashing`

Focus:
- recursion base case correctness
- bit tricks and binary representations
- linked structure manipulation
- monotonic stack/queue mindset
- map/set based counting and lookup

Target:
- 150-170 problems
- reduce silly bugs (null checks, pointer updates, bounds)

## Stage 3 - Pattern Layer (Weeks 9-11)

Topics:
- `13_Two_Pointers`
- `14_Sliding_Window`

Focus:
- turning O(n^2) into O(n)
- maintaining moving constraints
- left/right pointer invariants

Target:
- 50+ problems
- medium pattern questions in 25-35 mins

## Stage 4 - Trees, Graphs, and Greedy (Weeks 12-16)

Topics:
- `15_Trees`
- `16_BST`
- `17_Heap`
- `18_Graphs`
- `19_Backtracking`
- `20_Greedy`

Focus:
- DFS/BFS traversal confidence
- heap-based optimization
- decision tree pruning
- greedy proof intuition

Target:
- 140-160 problems
- confidently implement BFS/DFS/heap from scratch

## Stage 5 - Dynamic Programming Core (Weeks 17-20)

Topics:
- `21_Dynamic_Programming`
- `22_Trie`
- `23_Segment_Tree`

Focus:
- state definition and transition writing
- iterative vs recursive DP
- query data structures

Target:
- 70-90 problems
- medium DP in <=45 mins, hard DP with clear approach notes

## Stage 6 - Advanced Competitive Topics (Weeks 21-26)

Topics:
- `24_Advanced_Graphs`
- `25_Competitive_Programming`
- `26_Number_Theory`
- `27_Geometry`
- `28_Game_Theory`
- `29_String_Algorithms`
- `30_DP_On_Trees`
- `31_DP_On_Graphs`

Focus:
- shortest paths / graph optimization
- theorem-driven implementation
- advanced string and DP applications

Target:
- 180+ problems
- readiness for contests and difficult interview rounds

## Weekly Execution Plan

1. 5 solving days
2. 1 revision day
3. 1 mock test day

Daily split:
- 2 easy
- 4 medium
- 1 hard or reattempt

## Revision Cycle

1. Revisit solved set after 3 days.
2. Revisit again after 10-14 days.
3. Revisit difficult set before interviews.

Tag your problems:
- `R1`: solved with hints
- `R2`: solved independently
- `R3`: can explain and code quickly

## Interview Readiness Checkpoints

Checkpoint A (after Stage 2):
- Arrays/Strings/Hashing/Binary Search solid

Checkpoint B (after Stage 4):
- Trees/Graphs/Greedy/Backtracking interview ready

Checkpoint C (after Stage 6):
- can handle hard graph + DP + theory combos

## Final 2-Week Sprint Before Interviews

1. Mixed-topic daily sets (8-10 questions)
2. Timed coding rounds (90 mins)
3. Recode top 30 weak problems from scratch
4. Practice verbal explanation of approach and complexity
